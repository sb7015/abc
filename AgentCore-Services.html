AgentCore Services Present in Architecture

6 of 7 AgentCore services are used in our architecture.

AgentCore Runtime

Hosts our entire LangGraph orchestration (Multi-Hop, Router, Retriever, Generator, Validator) as a single deployment unit. Each user session gets its own isolated microVM with dedicated CPU, memory, and filesystem. Scales automatically based on demand.

Benefit: Zero infrastructure management. Complete session isolation between users. Auto-scaling without manual pod tuning. Supports real-time and long-running workloads up to 8 hours.

AgentCore Memory

Two-tier managed memory. Short-term memory stores raw conversation events within a session. Long-term memory asynchronously extracts structured facts, summaries, and preferences using an LLM and persists them across sessions. We use the semantic strategy.

Benefit: Agent remembers context within and across sessions without custom storage code. Semantic search retrieves relevant memories in ~200ms. Extraction runs in background without blocking the user.

AgentCore Gateway

MCP

Wraps external services (Amazon Bedrock, OpenSearch, Cohere Reranker) into MCP-compatible tools that agents can discover and use. All agent tool calls are routed through Gateway, which enables policy enforcement and automatic logging on every call.

Benefit: Centralized control over all external calls. Swap a backend service (e.g., change vector store) by updating Gateway registration, not agent code. Every call is traceable and policy-checked.

AgentCore Identity

Authenticates users against enterprise identity providers (Okta, Microsoft Entra ID, Amazon Cognito). Generates and manages Session IDs. Handles OAuth delegation so agents can act on behalf of authenticated users with proper permissions.

Benefit: Managed auth integration with enterprise IdPs. Session ID generation ties the entire request lifecycle together (Runtime, Memory, Observability). Supports OAuth delegation for accessing user-scoped resources.

AgentCore Policy

Defines guardrails in natural language that are enforced deterministically at the Gateway level. Rules execute as code, not as LLM instructions. Cannot be bypassed by creative prompting. Enforced on every tool call before it reaches the external service.

Benefit: Deterministic guardrails outside agent code. Division access control, topic boundaries, and response filtering enforced at infrastructure level. Critical for regulated enterprise environments.

AgentCore Observability

Wraps the entire Runtime automatically. Captures full traces of every request lifecycle from Memory Retrieval through Validation. Logs latency, token usage, tool call success/failure, and error details for every step and every agent in the pipeline.

Benefit: Complete visibility into agent behavior with zero instrumentation code. Debug failed responses by pulling the exact trace: what was retrieved, generated, and why the Validator rejected it.
