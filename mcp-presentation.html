Model Context Protocol (MCP)

Standardizing AI Tool Integration

What is MCP and Why It Exists

Definition of Model Context Protocol

MCP is a standardized protocol that allows Large Language Models (LLMs) to securely connect with external tools, data sources, and services.

The "Parrot Problem" with LLMs

  • LLMs can only repeat information from training data
  • No access to real-time data or external systems
  • Cannot perform actions beyond text generation
  • Limited to static knowledge cutoffs

Need for external tool integration is critical!

MCP as "USB-C for AI Applications"

  • One protocol works with any compatible tool
  • No custom integrations needed for each app-tool combination
  • Plug-and-play functionality for AI applications

Standardization Benefits:

  • Reduced development time
  • Consistent interfaces
  • Better security and reliability

Basic Architecture (Client-Server-Host)

Core Components

üîß MCP Server

Contains tools and capabilities

  • Hosts specific tools
  • Defines capabilities
  • Handles authentication

üì± MCP Client

Applications using tools

  • AI applications
  • Discovers tools
  • Sends requests

üè† Host

Environment where clients run

  • Manages connections
  • Process orchestration
  • Security boundaries

Communication Flow

1. Client discovers available tools from Server
2. Client requests tool execution with parameters
3. Server validates request and executes tool
4. Server returns results to Client
5. Host manages the entire process securely

Understanding JSON-RPC 2.0 - Basics

JSON-RPC 2.0 Basics

JSON-RPC 2.0 is a lightweight remote procedure call protocol that allows programs to call functions on remote systems.

Key Components

  • Request: Calls a method with parameters
  • Response: Returns result or error
  • Notification: One-way message (no response expected)

Basic Request Structure

Template:

{ "jsonrpc": "2.0", "method": "tool_name", "params": { "arg1": "value1", "arg2": "value2" }, "id": "unique_request_id" }

üçï Real Example - Pizza Order:

{ "jsonrpc": "2.0", "method": "order_pizza", "params": { "size": "extra_large", "toppings": ["pepperoni", "mushrooms"], "crust": "stuffed", "extra_cheese": true }, "id": "order_12345" }

Basic Response Structure

Template:

{ "jsonrpc": "2.0", "result": { "output": "Tool execution result" }, "id": "unique_request_id" }

üçï Pizza Order Response:

{ "jsonrpc": "2.0", "result": { "order_confirmed": true, "estimated_time": "30 minutes", "total_price": "$24.99", "tracking_url": "pizza.com/track/12345" }, "id": "order_12345" }

Schema Examples Explained

The game character schema shows how schemas can:

  • Use patterns (only letters/numbers in name)
  • Set min/max lengths (name 3-20 characters)
  • Create nested objects (stats within character)
  • Provide smart defaults (everyone starts with wooden sword!)
